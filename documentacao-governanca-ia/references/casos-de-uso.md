# Casos de uso

Casos de uso conectam usuario, regra, dado, permissao e resultado esperado.

## Para cada caso de uso, documente
- Nome.
- Ator principal.
- Objetivo.
- Pre-condicoes.
- Dados de entrada.
- Validacoes obrigatorias.
- Permissoes necessarias.
- Fluxo principal.
- Fluxos alternativos.
- Erros esperados.
- Dados retornados.
- Tabelas, APIs ou servicos envolvidos.
- Testes recomendados.

## Exemplos de casos comuns
- Criar conta.
- Completar perfil.
- Configurar preferencias.
- Buscar item.
- Encontrar match.
- Editar dados proprios.
- Acessar admin.
- Editar dado global.
- Reportar problema.

## Sinais de problema
- Caso de uso sem dono claro.
- Permissao definida apenas pela interface.
- Validacao duplicada e divergente.
- Erros nao tratados.
- Dados sensiveis retornados sem necessidade.
